# Migrate Docs

The idea is to take content from the scattered sources, decide what to keep,
discard or refine and then, create or update docs without duplication, in line
with our target structure.

## Process

1. Identify the source, one or a small set of files or pages
2. Identify the target: the new home(s) for the content
3. Identify duplication between the source and target
4. Decide what to discard/keep and what to refine
5. (Write a plan to) update or create the target, and delete the source

## Guidelines

- The content is what we care about
- The format should follow our target structure
- Use [`/docs-update-or-create`](../docs-update-or-create/SKILL.md)

## Output

Either the plan or (if instructed to) do the migration from step 5.
